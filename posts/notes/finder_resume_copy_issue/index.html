A Finder Resume Copy Issue

Published 17 August 2025

I hit a weird Finder copy failure the other day.

This was on macOS Sequoia. The copy failed halfway, and Finder said I could resume it. I didn’t really want to trust that.

Once a copy has already failed, it’s hard to know if the destination is still clean enough to trust. For this case, deleting the destination file or folder and starting the copy again from scratch felt like the safer option.

I wouldn’t call this a confirmed Finder bug from one case. Could’ve been file corruption, file permissions, or some other weird edge case. Still odd enough that I wanted to write it down.
