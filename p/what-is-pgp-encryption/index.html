什么是 PGP 加密以及它如何工作？

Aug 13, 2019
1 minute read

当你使用 ProtonMail 发送加密电子邮件时，你的信息将会使用 PGP 加密进行保护。关于什么是 PGP，本文解释了我们安全承诺背后的技术。

PGP 表示 Pretty Good Privacy，但这个名字是一个讽刺性的保守说法。事实上，PGP 是世界上使用最广泛的电子邮件加密系统。当你使用 PGP 加密发送信息时，没有人可以在发送过程中截取和阅读你的消息。PGP 在过去的几十年的使用中已经经过了全面的现场测试，它的几个漏洞是总所周知的，并且它于其他加密客户端具有广泛的兼容性。出于这些原因，我们使用 PGP 作为我们安全架构的支柱。

本文是解释 ProtonMail 背后的一些技术的系列文章的一部分。我们已经介绍了端到端加密和零访问加密。在这里，我们将了解 PGP 是什么，它是如何工作的，以及如何使用它来保护你的通信。

什么是 PGP？

PGP 是一种加密方法，可以让我们在线私密交流。 使用 PGP 发送消息时，消息在通过互联网之前会在你的设备上转换为不可读取的密文。收件人只有密钥将密文转换回他们设备上的可读文本。PGP 除了验证发送者的身份，还会验证该消息在传输的过程中有没有被篡改。

在没有 PGP 之前，你的互联网提供商、电子邮件提供商、黑客或政府理论上可以阅读你的信息。PGP 由 Jon Callas 和 Phil Zimmerman 于 1991 年开发，让电子邮件和其他类型的信息进行私密交流。今天，PGP 已经进行标准化为 OpenPGP，使任何人都可以编写与其他使用兼容和互操作性的 PGP 软件。已经创建了几个符合 OpenPGP 的开发库，以帮助程序员在他们的应用程序中实现 PGP 加密。ProtonMail 是其中这两种库的维护者：OpenPGP.js，针对 JavaScript 编程语言（在我们的 web 应用程序中使用），和 GopenPGP，针对 Golang 编程语言（在我们的移动端和桌面应用程序中使用）。特别是 OpenPGP.js 是世界上使用最广泛的 OpenPGP 库之一，并且经过安全专家的全面审核。

从历史上看，PGP 很难使用，需要在电子邮件提供商或客户端的基础之上增加额外的应用程序。你还必须手动生成加密密钥并与联系人交换密钥。使用 ProtonMail，PGP 是内置的，并且自动运行且无形的。当你使用另一个 ProtonMail 用户编写电子邮件并发送时，邮件加密和签名将自动进行。你无需做任何事情或需要任何专业技术知识。ProtonMail 使 PGP 加密变得更简单、方便。

PGP 如何运作？

查看这张图来理解 PGP 加密是如何工作是非常有用的，PGP 使用对称密钥加密（即一次性会话密钥加密和解密消息）和公共密钥加密（即接收者独有的密钥加密和解密会话密钥）的组合。

对于本文，我们坚持高级概念。如果你对加密背后的数学感兴趣，你可以找到稍微简化的解释来消化这些。

PGP 加密

PGP 做的第一件事就是生成随机会话密钥(random session key) 。这个密钥是是一个巨大的数字，用于加密和解密消息的内容。只有知道会话密钥的人才能读取消息，而且消息太大无法猜测。会话密钥也不会再用于其他的消息。

接下来，会话密钥使用接收者的公钥加密 。公钥对于每个人来说都是独一无二的，意味着要共享。由于它没有变化，你的公钥就像一个电子邮件地址。它与你息息相关，任何人都可以用它向你发送加密信息。

每个人的公钥对应于他的私钥，这是保密的。在 PGP 中，当接收者收到加密邮件时，他们使用自己的私钥解密会话密钥。然后明文会话密钥解密该消息。

你可能想知道为什么 PGP 会采取额外的步骤来加密消息和会话密钥。这是因为公钥加密比对称加密慢得多，尤其对于大消息。使用公钥直接加密和解密大型电子邮件或文件需要花费太多时间和计算能力。但是没有公钥加密的对称加密就不太方便了，因为你需要以某种方式于接收者共享会话密钥。以明文方式这样是不安全的，并且通过另一个加密频道或亲自进行这样做是不切实际。因此，PGP 结合了对称加密的效率和公钥加密的便利性。

数字签名

PGP 还有两个方面需要注意。首先是数字签名。数字签名向接收者证明攻击者没有操纵消息和发送者。 它通过使用发送者私钥和明文消息的数学约简(称为消息摘要)的组合来创建唯一的数字(数字签名)来实现这一点。如果私钥或消息被更改，则数字签名无效。

信任公钥

数字签名有助于减轻复杂的攻击，但是发送者如何知道他们正在使用的公钥是属于他们认为属于的人呢？毕竟，服务器可以轻松地向发送者提供伪造公钥。

为了解决这个问题，我们引入了地址验证，它允许您共享您的公钥，并对您亲自验证的其他人的公钥进行数字签名。这些受信任的公钥然后被安全地存储在您的加密联系人中。

此外，我们正在进行一个名为关键透明度的项目。它将自动验证您向其发送电子邮件的每个收件人的公钥，而无需任何手动操作。一旦准备好，我们将发布一篇博客文章，详细介绍这个特性。

PGP 有多安全？

PGP 是经过实战检验的标准，我们几乎可以肯定，即使像 NSA 这样的情报机构也无法破解其加密。（PGP 是 Edward Snowden 在向 Glenn Greenwald 泄露机密文件时的首选加密方法）虽然 PGP 的某些实施存在安全漏洞，例如臭名昭着的 Efail 漏洞，但 PGP 本身非常安全。ProtonMail 尚未受到任何已知漏洞的影响。

像大多数其他信息安全系统一样，最大的弱点是用户。正如这幅漫画所示，最简单、最有效的攻击往往是最不高科技的。网络钓鱼仍然是最常见的一种网络攻击，如果您的设备或帐户受到威胁，PGP 无法保护您。(看看这些电子邮件安全提示)

如何使用 PGP 加密

就安全性而言，PGP 对用户来说并没有使事情变得更容易，因为程序是出了名的复杂。大多数不懂技术的人永远不会费心去学习如何使用它。知道如何使用它的人通常不知道，因为它太乏味了。虽然 Thunderbird、Outlook 和苹果邮件都有支持PGP加密的程序，但这些都不是日常电子邮件的实用解决方案。

ProtonMail 通过使 PGP 加密自动化和内置来解决这个问题。任何人都可以在一两分钟内创建一个免费的 ProtonMail 账户，并立即开始发送 PGP 电子邮件。ProtonMail 还可以提供高度隐私，因为我们不需要任何身份信息来创建帐户。

ProtonMail 还提供完整的 PGP 支持，允许高级用户在其邮箱中发送和接收来自非 ProtonMail 用户的 PGP 加密电子邮件。对于传统的 PGP 用户来说，这是迄今为止与其他 PGP 用户进行通信的最简单方式，同时还具有 ProtonMail 的便利性。按照链接了解如何将 ProtonMail用作PGP客户端。

最后的想法

随着超级计算机变得更快，新的加密标准越来越受欢迎，ProtonMail 将继续适应新的网络威胁和用户的需求。我们的密码学家正在努力开展新项目，如关键透明度和源代码透明度。我们最近还将用户升级为椭圆曲线加密，从而最大限度地提高了安全性和效率。 在我们社区的支持下，我们正在经过时间考验的基础上建立一个更安全的互联网。
