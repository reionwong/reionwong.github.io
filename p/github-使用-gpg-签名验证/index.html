Github 使用 GPG 签名验证

Aug 03, 2019
1 minute read

GPG 是一个用于数据加密和数字签名的程序，由于被广泛应用，后来形成了一个开放的标准 OpenPGP，而 GnuPG 是实现了该标准的一个开源程序，在 Linux 下可以使用 GnuPG 来管理密钥、加密解密文件、电子邮件、数字签名文件等内容。

在 Linux 下，一般可以在系统软件源中找到 GnuPG，比如 debian 发行版可以通过以下命令安装：

$ sudo apt install gnupg

GnuPG 简单用法

生成密钥对

$ gpg --gen-key

如果你要使用完整的生成选项，可以这样：

$ gpg --full-gen-key

查看密钥信息

查看公钥：

$ gpg --list-keys

可以加入 –keyid-format 参数：

$ gpg --list-keys --keyid-format LONG

查看密钥：

$ gpg --list-secret-keys

导出公钥：

$ gpg --armor --export 3AA5C34371567BD2

其中 “3AA5C34371567BD2” 是你的 GPG key ID，可以通过上一条查看本地密钥信息中获取到。

将 GPG 密钥添加到你的 GitHub 账户

GitHub 支持多种 GPG 密钥算法，如果你添加不受支持算法生成的密钥可能会遇到错误，以下是 Github 支持的密钥算法：

  • RSA
  • ElGamal
  • DSA
  • ECDH
  • ECDSA
  • EdDSA

在 GitHub 任意页面的右上角点击你的头像，弹出菜单选择 “Settings”，进入设置页面后在左侧栏找到 “SSH and GPG keys”，然后粘贴到 Key 的文本框里即可(GPG Public Key)。

使用 git 全局签名密钥

列出密钥列表：$ gpg --list-secret-keys --keyid-format LONG

从 GPG 密钥列表中，复制你要使用的 GPG 密钥 ID。在此示例中，GPG 密钥 ID为3AA5C34371567BD2：

$ gpg --list-secret-keys --keyid-format LONG
/Users/hubot/.gnupg/secring.gpg
------------------------------------
sec   4096R/3AA5C34371567BD2 2016-03-10 [expires: 2017-03-10]
uid                          Hubot 
ssb   4096R/42B317FD4BA89E7A 2016-03-10

接下来要在 git 中设置 GPG 签名密钥：

$ git config --global user.signingkey 3AA5C34371567BD2

如果要让当前git项目启用签名验证：

$ git config commit.gpgsign true

如果要让所有项目都启用签名验证：

$ git config --global commit.gpgsign true

tips: git 里填写的提交者信息，必须跟 GitHub 帐号一致的邮箱，否则会显示 “Unverified” 标志。

签名提交(Signing commits)

在本地分支中提交更改时，可以将 -S 参数加入 git commit 命令中：

$ git commit -S -m your commit message

签名 Tag 提交

要对标记进行签名，请添加 -s 到你的 git tag 命令中：

$ git tag -s xxx

验证签名 tag：$ git tag -v xxx

参考: https://help.github.com/en/articles/managing-commit-signature-verification
