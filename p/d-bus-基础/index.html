D-Bus 基础

Sep 29, 2018
2 minute read

D-Bus 是一种高级的进程间通信机制，是 freedesktop.org 的一个项目，为 Linux 桌面环境提供标准，是一个为应用程序间通信的消息总线系统, 主要用于在 Linux 进程之间的通信。

D-Bus 有三层架构的进程间的通信：

  • 接口层：由函数库 libdbus 提供，进程可通过该库使用 D-Bus
  • 总线层：由 D-Bus 总线守护进程提供，它在 Linux 系统启动时运行，负责进程间的消息路由和传递，包括 Linux 内核和 Linux 桌面环境的消息传递。
  • 包装层：基于特定应用程序框架的 Wrapper 库，比如 Gtk+、Qt 都有相应的包装层。

本文主要介绍 Qt DBus 的用法，Gtk+ 不会用。

应用

目前 D-Bus 已被大多数 Linux 桌面环境所使用，开发者可使用 D-Bus 实现各种复杂的进程间的通信任务，就像 DDE (Deepin Desktop Environment) 的后端用 Golang 语言来写的，使用 D-Bus 可以实现后端进程与前端进程的通信，后端进程 daemon 提供各种 D-Bus 服务，前端可通过后端提供的接口来进行对接，获取数据。

查看 D-Bus 总线上的服务和对象可以通过 d-feet、qdbusviewer 等工具。 发送信号可以使用 dbus-send，查看 D-Bus 消息流可以使用 dbus-monitor。

在 Qt 中使用 DBus

QtDBus 模块 是 Qt 对 DBus 底层 API 封装的实现，如果你使用 qmake 构建 Qt 程序，可以将下面一行添加到 .pro 文件中，让你的项目链接到 Qt DBus 库：

QT += dbus

cmake 可以这样（xxx是你的项目名称）

简单用法

不需要在 D-Bus 守护进程上注册服务和注册对象。

注册对象后，对象中导出的槽函数就可以供其他客户端来调用；注册了服务后对象就有了公共名，没有注册是只有三个唯一名。

在服务端只需要创建一个信号源后发送即可：

1). 创建 Qt DBus 信号

QDBusMessage msg = QDBusMessage::createSignal("/rekols/path", "rekols.interface");

2). 给信号赋值

msg << QString("Hello World");

3). 发射信号

QDBusConnection::sessionBus().send(msg);

在客户端中：

1). 绑定信号

QDBusConnection::sessionBus().connect("", "", "dbus.client.interface", "Active", this, SLOT(ActiveEvent(QString)));

在 ActiveEvent 中执行你需要对这个信号做出的反应即可。

实现单进程程序

Qt 要实现单进程应用程序其实很简单，特别是用 DBus 来实现，首先连接到 DBus 总线守护程序，然后尝试注册一个服务，如果注册失败了，说明已经运行了一个进程则直接 return。

但是这样的做法只对 Linux 平台有效，要想跨平台还是通过 QLocalSocket/QLocalServer 监听服务连接吧 (Windows、Linux 都适用)。

#include "mainwindow.h"
#include <QApplication>
#include <QDBusConnection>
#include <QDBusInterface>

int main(int argc, char *argv[])
{
    QApplication a(argc, argv);

    // 连接到 DBus 总线
    QDBusConnection dbus = QDBusConnection::sessionBus();

    // 如果注册成功则创建窗口(必须以 xxx.xxx 来指定 service name)
    if (dbus.registerService("com.deepin.xxxxx")) {
        MainWindow w;
        w.show();

        return a.exec();
    }

    // 注册失败会退出程序
    return 0;
}
